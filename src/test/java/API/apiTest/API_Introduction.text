• It acts like a messenger that takes requests from a client and returns responses from a server.

--------------------What Swagger is (simple)-------------------
Swagger is basically an API manual + testing page.
It shows:All endpoints (GET, POST, PUT, DELETE)
Request body format
Headers (auth tokens, etc.)
Status codes
1️⃣ Developers create Swagger
Backend developers generate Swagger/OpenAPI docs when they build APIs.
2️⃣ QA uses Swagger
QA engineers use Swagger to: See endpoints
understand request body
check headers
verify response format
test status codes

-----------Postman----------
It helps to test REST API urls
Its popular for developers and QA
We can use manual and automation
It uses Chai library to write test cases

----------Headers---------------
It's part of URL. Additional information about request and response
It works like a map key&value structure
